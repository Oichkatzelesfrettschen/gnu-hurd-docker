================================================================================
ARCHITECTURAL RECOMMENDATIONS
================================================================================

CURRENT ARCHITECTURE (Single-Container Monolith)
  - QEMU process inside Docker container
  - User-mode networking (SLIRP) with port forwards
  - Local volume (bind mount) for QCOW2 disk
  - Appropriate for: Single-developer workstation, development

RECOMMENDED FOR PRODUCTION (Multi-Container Stack)
  - StatefulSet for VMs (Kubernetes or Swarm)
  - Observability sidecars (Prometheus, Loki)
  - Load balancer (Traefik or Nginx)
  - Shared storage (NFS, Ceph, or cloud block)
  - Secrets manager (Vault or cloud-native)

SCALABILITY ASSESSMENT
================================================================================

Current Limitations:
  - Single host only (no Docker Swarm/Kubernetes)
  - No horizontal scaling (1 VM max)
  - No high availability (single point of failure)
  - No service discovery or load balancing

Recommended Path:
  Phase 1: Docker Swarm (3-6 months)
    - Easy migration from Compose
    - Multi-host support
    - Basic orchestration

  Phase 2: Kubernetes (6-12 months)
    - Cloud-native deployment
    - Advanced scheduling
    - Full ecosystem access

Performance:
  - KVM: 30-60s boot, excellent runtime performance
  - TCG: 3-5 min boot, acceptable for development
  - Network: 500-900 Mbps with e1000/SLIRP
  - Disk: 100+ MB/s with IDE/writeback cache
